# コードレビュー

現在開いているファイルまたは指定されたコードに対して、以下の観点でコードレビューを行ってください。

## レビュー観点

1. **セキュリティ**: 脆弱性や不適切なデータ処理がないか
2. **パフォーマンス**: 不要な再レンダリング、重い計算、N+1クエリなど
3. **型安全性**: TypeScriptの型の適切な使用
4. **アクセシビリティ**: a11yに配慮した実装か
5. **プロジェクト規約**: ESLint、oxlint、oxfmtに準拠しているか

## 出力形式

- 日本語でフィードバックを出力する
- 問題点があれば具体的な修正案を提示する
- 良い点も簡潔に指摘する
